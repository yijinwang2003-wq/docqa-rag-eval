Key findings
01

Hybrid retrieval consistently improved answer relevancy by 2–4 points over dense-only retrieval across both chunking strategies. BM25 keyword matching anchors retrieval to exact technical terms like chunk_overlap and RecursiveCharacterTextSplitter that dense search drifts away from.

02

Fixed chunking outperformed semantic chunking on context precision (0.710–0.720 vs 0.682–0.690). Semantic chunks on structured technical documentation tend to be larger and less uniform, introducing irrelevant context that hurts ranking quality.

03

Corpus quality mattered more than retrieval tuning on a noisy sitemap corpus. Filtering, boilerplate cleaning, and a guaranteed core URL allowlist were necessary preconditions before configuration differences became statistically meaningful.

04

Faithfulness saturated near 1.0 across all configurations. With an explicit grounding prompt, the model stayed within retrieved context regardless of retrieval strategy. Answer relevancy and context precision are the informative signals for comparing retrieval configurations in this domain.
